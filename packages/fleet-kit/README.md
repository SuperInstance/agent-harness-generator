# @superinstance/fleet-kit

> SuperInstance fleet protocol SDK — plug your harness into the fleet.

A TypeScript SDK that makes any agent harness generated by the [agent-harness-generator](https://github.com/SuperInstance/agent-harness-generator) a first-class participant in the SuperInstance fleet.

## Why?

Standalone harnesses are islands. Fleet-kit gives them:

- **A2A orchestration** — spawn subagents via OpenClaw sessions API
- **I2I shared state** — read/write bottles via harbor-daemon protocol
- **GC reporting** — report disk/RAM/load to the fleet GC ledger
- **Fleet registry** — register/unregister, heartbeat
- **Pulse metrics** — push conservation metrics (γ/η/c ratio)

## Install

```bash
npm install @superinstance/fleet-kit
```

## Usage

```typescript
import { A2AClient, I2IClient, GCClient, FleetRegistry, PulseClient } from '@superinstance/fleet-kit';

// Spawn a subagent
const a2a = new A2AClient();
const result = await a2a.spawn({ task: 'Analyze the data' });

// Write a bottle to harbor
const i2i = new I2IClient();
await i2i.writeBottle('analysis-result', 'The data shows...');

// Report GC metrics
const gc = new GCClient({ instanceName: 'my-harness' });
await gc.report();

// Register with fleet
const registry = new FleetRegistry();
await registry.register({
  name: 'my-harness',
  version: '1.0.0',
  tier: 'hot',
  host: 'oracle2',
  capabilities: ['analysis', 'reporting'],
});

// Push pulse metrics
const pulse = new PulseClient({ instanceName: 'my-harness' });
pulse.setServiceCount(3);
await pulse.push();
```

## When used with `create-agent-harness`

Generated harnesses get all fleet protocol files automatically:

```
my-harness/
├── AGENTS.md           # Fleet constitution
├── .gcconfig           # GC policy manifest
├── FLEET_PROTOCOL.md   # Integration docs
├── i2i-vessel/         # Shared state directory
├── scripts/
│   ├── fleet-scout.sh  # Fleet registration & heartbeat
│   └── gc-self-audit.sh # Local GC reporting
└── install-openclaw.sh # Full install + fleet setup
```

## API

### A2AClient

| Method | Description |
|--------|-------------|
| `spawn(spec)` | Spawn a subagent to complete a task |
| `send(sessionKey, message)` | Send a message to a running subagent |

### I2IClient

| Method | Description |
|--------|-------------|
| `writeBottle(name, content)` | Write a bottle to harbor (or local filesystem) |
| `readBottles()` | Read all bottles |
| `readSessionState()` | Read SESSION-STATE.md |
| `writeSessionState(content)` | Write SESSION-STATE.md |

### GCClient

| Method | Description |
|--------|-------------|
| `gatherMetrics()` | Gather current system metrics |
| `report()` | Report metrics to fleet GC ledger |

### FleetRegistry

| Method | Description |
|--------|-------------|
| `register(spec)` | Register with the fleet |
| `heartbeat(name)` | Send a heartbeat |
| `deregister(name)` | Deregister from the fleet |

### PulseClient

| Method | Description |
|--------|-------------|
| `snapshot()` | Take a pulse snapshot |
| `push()` | Push metrics to conservation-meter |
| `setServiceCount(n)` | Set active service count |
